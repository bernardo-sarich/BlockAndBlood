# TASK-11 — Criterios de Éxito del MVP

**Sección GDD:** 13. Criterios de éxito del MVP

---

## Descripción
Checklist de validación técnica (mes 4) y criterio de diseño (mes 5) que determinan si el MVP está completo y si el juego merece continuar hacia la versión completa.

---

## Criterios técnicos — Mes 4

- [ ] Una run completa de ~15 minutos sin crashes
- [ ] El pathfinding recalcula correctamente en tiempo real sin stutters visibles
- [ ] Las 15 cartas tienen efectos que funcionan correctamente
- [ ] El boss completa sus 2 fases correctamente
- [ ] El juego corre a **60 fps estables** en hardware de gama media

---

## Criterio de diseño — Mes 5

**Al menos 3 de 5 testers externos, sin explicación previa, quieren hacer otra run inmediatamente después de terminar la primera.**

Este es el único criterio que importa para decidir si continuar el desarrollo hacia la versión completa.

---

## Métricas secundarias de playtesting

| Pregunta | Objetivo |
|----------|---------|
| ¿El jugador entiende el pathfinding solo? (sin explicación) | Sí |
| ¿El jugador cambia su estrategia después de perder vidas con los Blindados? | Sí |
| ¿El jugador recuerda qué cartas eligió al terminar la run? | Sí |
| ¿El jugador comenta algo sobre las sinergias entre cartas? | Sí |

Si las respuestas a estas cuatro preguntas son mayormente **sí**, el sistema funciona como fue diseñado.
